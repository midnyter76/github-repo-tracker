<!-- CONTRACT this plan DEFINES — Plan 03 (report) renders this exact shape. -->
compute_buckets(snapshots_dir, metadata_path, now) -> dict with four fixed keys, uniform shape:
```python
{
  "brand_new_weekly":  {"active": True,  "snapshots_available": N, "window_target": 7,  "entries": [<entry>, ...]},  # <= 10
  "brand_new_monthly": {"active": True,  "snapshots_available": N, "window_target": 30, "entries": [<entry>, ...]},  # <= 5
  "spike_24h":         {"active": bool,  "snapshots_available": N, "window_target": 2,  "entries": [<entry>, ...]},   # <= 10
  "velocity_30d":      {"active": bool,  "snapshots_available": N, "window_target": 30, "entries": [<entry>, ...]},   # <= 10
}
```
<entry> (all four buckets identical shape; velocity ALWAYS expressed per-day):
```python
{ "id": "<str>", "full_name": "owner/repo", "html_url": "https://...",
  "description": "<raw str — sanitized later at render>", "created_at": "<ISO str>",
  "stars": <int>, "velocity_per_day": <float> }
```
new-repo buckets: `active` is always True. breakthrough buckets: `active=False` => entries == [] and the caller renders the warming note from snapshots_available / window_target.

Token handling stays Phase-1 only: rank.py makes no API calls and references no secret.
