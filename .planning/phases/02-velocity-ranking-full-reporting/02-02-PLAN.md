<!-- CONTRACT this plan DEFINES — Plan 03 renders markers, Plan 04 wires ordering. -->
classify_and_update(seen: dict, reported_ids: list[str], report_date: str) -> tuple[dict, dict]
  returns (markers, updated_seen) where:
    markers[rid] == "new"        for rid not in seen
    markers[rid] == "returning"  for rid already in seen
    updated_seen == copy of seen + {rid: {"first_seen": report_date}} for each NEW rid
  The input `seen` dict is NOT mutated. `updated_seen` is written by the caller AFTER the report (D-10).
</interfaces>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Denial of Service | load_seen on corrupt json | mitigate | try/except JSONDecodeError -> warnings.warn + {} (mirrors store.py); a corrupt ledger degrades to "everything looks new this run", never a crash. |
| T-02-05 | Spoofing | repo identity continuity | mitigate | Keys are str(repo.id), never full_name — a transferred/renamed repo keeps its 🆕/↩ history (D-09). Enforced by acceptance criteria + test. |
| T-02-06 | Tampering | premature store write | mitigate | classify_and_update never writes; it returns `updated` for the caller to persist AFTER the report (D-10), so a crash mid-run cannot mark a repo seen without it appearing in a digest. |

seen.py makes no API calls and references no secret; token handling remains Phase-1 only.
