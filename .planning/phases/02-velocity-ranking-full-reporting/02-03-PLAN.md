<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo `description` -> reports/*.md (public) | THE primary Phase 2 threat. Descriptions are attacker-influenceable GitHub API text rendered into a public markdown file in a browsable repo (ASVS V5). |
| repo `full_name` -> link label | full_name is used as a link *label* `[{full_name}]({html_url})`; it is trusted GitHub-issued `owner/repo` and not re-parsed — only description needs sanitization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | description with embedded newline/control chars | mitigate | sanitize_description collapses \n/\r/\t and strips control chars before rendering, preserving the D-04 single-line invariant. Asserted by test. |
| T-02-08 | Tampering | description containing markdown-link syntax `](url)` | mitigate | sanitize_description neutralizes `[` `]` and guarantees the output contains no `](` substring — a description cannot inject a clickable link. Asserted by test. |
| T-02-09 | Tampering | description containing raw HTML `<...>` / backticks | mitigate | sanitize_description strips `< >` and backticks; truncates to DESCRIPTION_MAX_CHARS to bound output. |

Deeper star-gaming / fake-velocity filtering is out of scope (Phase 3 / HARD-03). No secrets are read or written here; the token never reaches the digest.
</threat_model>
