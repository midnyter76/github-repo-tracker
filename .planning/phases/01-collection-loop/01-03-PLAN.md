| T-01-08 | Tampering / data loss | same-day re-run truncating snapshot | mitigate | Load-then-upsert merge (`{**existing, **new}`) guarantees a retry can only add/update, never drop entries (DATA-04 / Pitfall 5); covered by an explicit idempotency test. |
| T-01-09 | Tampering | untrusted repo description text in JSON | accept | `json.dumps` escapes all string content; Phase 1 never interpolates it into shell or markdown. Phase 2 owns any rendering-time escaping. |
