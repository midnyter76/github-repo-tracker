Gmail-safe substitution rules already established in this codebase (do not violate):
- NEVER `display:flex`, `display:grid`, `gap:` anywhere in output.
- Multi-column rows use `<table role="presentation"><tr><td width="X%">...` with
  `border-spacing` for gutters (see existing grid_html/strip_html tables).
- For the reference's `justify-content:space-between` (kicker pinned top, body pinned
  bottom, regardless of body length): use a nested `<table role="presentation"
  width="100%" height="46">` with two `<tr>` rows — row 1 `<td valign="top">` = kicker,
  row 2 `<td valign="bottom">` = body (empty-state text OR active value/name block).
  46px = 64px card min-height minus 2×9px vertical padding. Apply this nested table
  identically in BOTH the active and empty branches (same wrapper, different body content)
  so both render at the same height with content anchored the same way.
- For the reference's two-line-on-one-baseline patterns (value+unit, strip value+label):
  plain inline `<span>` elements naturally sit on one line/baseline — no flex needed,
  just put them adjacent in the markup with a small `margin-left` on the second span for
  the gap.
- For the reference's single bordered/rounded joined strip: put `border`, `border-radius`,
  `background` on the OUTER `<table>` element itself (not per-cell), and use `border-right`
  on the first N-1 `<td>` cells as internal dividers. `overflow:hidden` isn't needed since
  there's no flex to clip.

NEW-dot marker: reuse the existing marker rule verbatim (already used in `render_html_row`
and `_count_brand_new`): `markers.get(str(entry["id"]), "new") == "new"`. This is a NEW
per-leader-card indicator — currently leader cards show no NEW badge at all.

Intentional deviations from the reference (do NOT "fix" these — they are pre-existing,
test-pinned product decisions from an earlier quick task):
- BRAND NEW stat stays color `#f4f4f5` (white), not green `#34d399` like the reference.
- Stats-strip label stays `TOP */DAY` (asterisk), not `TOP ★/DAY` (star glyph) — an
  existing test pins the asterisk text.

Security: repo name escaping (`html.escape(name, quote=True)`) must be preserved exactly
as-is — do not touch or weaken it.
